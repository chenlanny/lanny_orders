# 管理後台功能說明
**實現日期**: 2026年4月3日

---

## 功能概覽

系統已完成三大管理功能模塊，提供完整的店家、菜單和揪團管理能力。

### 1. 店家管理 (`/admin/stores`)

#### 功能特性
- ✅ 查看所有店家列表
- ✅ 新增店家
- ✅ 編輯店家資訊
- ✅ 刪除店家（有訂單記錄的無法刪除）

#### 欄位說明
- **店家名稱**: 必填，例如：50嵐、清心福全
- **類別**: 必填，選擇「飲料」或「便當」
- **電話**: 選填，店家聯絡電話
- **地址**: 選填，店家地址

#### 後端 API
```
GET    /api/stores            - 取得所有店家
GET    /api/stores/{id}       - 取得單一店家詳情
POST   /api/stores            - 創建店家（僅管理員）
PUT    /api/stores/{id}       - 更新店家（僅管理員）
DELETE /api/stores/{id}       - 刪除店家（僅管理員）
```

---

### 2. 菜單管理 (`/admin/menu`)

#### 功能特性
- ✅ 選擇店家查看菜單
- ✅ 新增菜單項目
- ✅ 編輯菜單項目
- ✅ 刪除菜單項目（有訂單記錄的會設為下架）
- ✅ 切換上架/下架狀態

#### 欄位說明
- **品項名稱**: 必填，例如：珍珠奶茶
- **類別**: 必填，例如：奶茶、綠茶、主菜
- **價格規則**: 必填，JSON格式
  ```json
  {"中杯":50,"大杯":60,"瓶裝":65}
  ```
- **選項**: 選填，JSON格式，包含甜度、溫度、加料
  ```json
  {
    "sweetness": ["正常","小糖","微糖","半糖","少糖","無糖"],
    "temperature": ["正常冰","少冰","微冰","去冰","溫","熱"],
    "toppings": [
      {"name":"珍珠","price":10},
      {"name":"波霸","price":10}
    ]
  }
  ```
- **是否上架**: 控制項目是否顯示給用戶

#### 後端 API
```
GET    /api/stores/{storeId}/menu       - 取得店家可用菜單
GET    /api/stores/{storeId}/menu/all   - 取得所有菜單（含下架，僅管理員）
POST   /api/stores/{storeId}/menu       - 創建菜單項目（僅管理員）
PUT    /api/stores/menu/{menuItemId}    - 更新菜單項目（僅管理員）
DELETE /api/stores/menu/{menuItemId}    - 刪除菜單項目（僅管理員）
```

---

### 3. 揪團管理 (`/admin/events`)

#### 功能特性
- ✅ 查看所有進行中的揪團
- ✅ 展開查看每個揪團的店家列表
- ✅ 提早設定店家截止時間
- ✅ 查看各店家狀態（可點餐/已截止）

#### 使用場景
1. **突發狀況提早截止**: 店家臨時通知提早截單
2. **主動控制**: 揪團發起人可以主動控制各店家截止時間
3. **狀態監控**: 一目瞭然查看各店家的訂購狀態

#### 限制規則
- ✅ 只能提早截止時間，不能延後
- ✅ 不能設定過去時間
- ✅ 只有揪團發起人可以更新截止時間
- ✅ 活動已結束的無法更新

#### 後端 API
```
PUT /api/events/groups/{groupId}/deadline - 更新訂單群組截止時間（僅發起人）

Request Body:
{
  "deadline": "2026-04-03T18:00:00Z"
}
```

## 權限控制

### 管理員權限
- 店家管理：完整 CRUD 權限
- 菜單管理：完整 CRUD 權限
- 揪團管理：管理員可訪問，但僅發起人可修改截止時間

### 一般用戶
- 無法訪問管理後台
- 嘗試訪問會自動跳轉至首頁

## 資料驗證

### 後端驗證（DTO）
- **CreateStoreRequestDto**: 店家名稱、類別為必填
- **CreateMenuItemRequestDto**: 品項名稱、類別、價格規則為必填
- **UpdateOrderGroupDeadlineDto**: 截止時間為必填，且需通過業務邏輯驗證

### 前端驗證（Form）
- 所有必填欄位即時驗證
- JSON格式驗證（價格規則、選項）
- 日期時間驗證（不可選擇過去時間）

## 使用流程

### 店家設定流程
1. **新增店家** → 管理後台 → 店家管理 → 新增店家
2. **設定菜單** → 管理後台 → 菜單管理 → 選擇店家 → 新增菜單
3. **調整狀態** → 編輯菜單項目 → 切換上架/下架

### 揪團管理流程
1. **查看揪團** → 管理後台 → 揪團管理
2. **展開查看** → 點擊表格展開查看店家列表
3. **提早截止** → 點擊「提早截止」→ 選擇新時間 → 確認

⚠️ **權限隔離**: 管理功能僅限管理員，一般用戶無法訪問
⚠️ **操作記錄**: 所有操作都會記錄 Log（後端）

✅ 新增店家後立即設定菜單
✅ 測試菜單項目後再設為上架

✅ JSON格式錯誤會即時提示，請檢查格式
