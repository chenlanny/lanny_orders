# 繁體中文編碼說明

## 🌐 重要提醒
**本專案所有中文內容必須使用繁體中文（UTF-8）編碼，確保不會出現亂碼。**

---

## 📊 資料庫操作

### 1. 插入中文資料
執行SQL時必須使用 `-f 65001` 參數指定UTF-8編碼：

```bash
sqlcmd -S 192.168.207.52 -d L_DBII -U reyi -P "ReyifmE" \
  -i insert_menu_utf8.sql -f 65001
```

### 2. 查詢中文資料
查詢時也要加上 `-f 65001` 參數：

```bash
sqlcmd -S 192.168.207.52 -d L_DBII -U reyi -P "ReyifmE" \
  -Q "SELECT Name FROM MenuItems" -f 65001 -W
```

### 3. SQL腳本編寫規則
- ✅ 所有中文字串前必須加 `N` 前綴（表示Unicode字串）
- ✅ 範例：`INSERT INTO MenuItems (Name) VALUES (N'珍珠奶茶');`
- ✅ 文件必須保存為 UTF-8 編碼
- ❌ 錯誤：`INSERT INTO MenuItems (Name) VALUES ('珍珠奶茶');` ← 缺少N前綴

## ❓ 常見問題

### Q: 為什麼資料庫看到亂碼？
**A:** 執行sqlcmd時忘記加 `-f 65001` 參數

**解決方法：**
```bash
# 錯誤 ❌
sqlcmd -S ... -Q "SELECT Name FROM MenuItems"

# 正確 ✅
sqlcmd -S ... -Q "SELECT Name FROM MenuItems" -f 65001
```

---

### Q: 插入資料後變成 ??? 或亂碼？
**A:** SQL字串前忘記加 `N` 前綴

**解決方法：**
```sql
-- 錯誤 ❌
INSERT INTO MenuItems (Name) VALUES ('珍珠奶茶');

-- 正確 ✅
INSERT INTO MenuItems (Name) VALUES (N'珍珠奶茶');
```

---

### Q: API返回中文正常，但前端顯示亂碼？
**A:** 檢查 index.html 的 charset 設定

**解決方法：**
```html
<!-- 確保有以下設定 -->
<meta charset="UTF-8">
```

---

### Q: VS Code編輯SQL檔案時中文變亂碼？
**A:** 檔案編碼設定錯誤

**解決方法：**
1. 開啟檔案
2. 右下角點擊編碼（如：UTF-8）
3. 選擇「透過編碼重新開啟」
4. 選擇 UTF-8
5. 重新儲存檔案
