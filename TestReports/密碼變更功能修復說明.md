# 密碼變更功能修復說明

## 問題原因

**前端Settings.jsx只是顯示成功訊息，但並沒有實際呼叫後端API**，而且**後端也沒有密碼變更的API端點**。

這就是為什麼：
- 畫面顯示「密碼變更成功」
- 但登出後舊密碼仍然可以登入
- 因為密碼根本沒有被變更

## 修復內容

### 1. 後端新增密碼變更API

**新增檔案/修改：**

#### AuthDtos.cs
新增 `ChangePasswordRequestDto`：
```csharp
public class ChangePasswordRequestDto
{
    [Required(ErrorMessage = "目前密碼為必填")]
    public string OldPassword { get; set; } = string.Empty;

    [Required(ErrorMessage = "新密碼為必填")]
    [StringLength(100, MinimumLength = 6, ErrorMessage = "密碼長度必須在6-100字之間")]
    public string NewPassword { get; set; } = string.Empty;
}
```

#### AuthService.cs
新增 `ChangePasswordAsync` 方法：
```csharp
public async Task<bool> ChangePasswordAsync(int userId, ChangePasswordRequestDto request)
{
    // 1. 查詢用戶
    var user = await _context.Users.FindAsync(userId);
    if (user == null) return false;

    // 2. 驗證舊密碼
    if (!BCrypt.Net.BCrypt.Verify(request.OldPassword, user.PasswordHash))
        return false;

    // 3. 更新密碼（使用BCrypt Hash）
    user.PasswordHash = BCrypt.Net.BCrypt.HashPassword(request.NewPassword);
    await _context.SaveChangesAsync();

    return true;
}
```

#### AuthController.cs
新增 `/api/auth/change-password` 端點：
```csharp
[Authorize]
[HttpPost("change-password")]
public async Task<ActionResult<ApiResponse<object>>> ChangePassword([FromBody] ChangePasswordRequestDto request)
{
    // 從JWT Token取得UserId
    var userIdClaim = User.FindFirst(ClaimTypes.NameIdentifier);
    if (userIdClaim == null || !int.TryParse(userIdClaim.Value, out int userId))
        return Unauthorized(ApiResponse<object>.Fail("無效的使用者驗證"));

    var result = await _authService.ChangePasswordAsync(userId, request);
    
    if (!result)
        return BadRequest(ApiResponse<object>.Fail("目前密碼錯誤"));

    return Ok(ApiResponse<object>.Ok(null, "密碼變更成功"));
}
```

### 2. 前端呼叫API

**Settings.jsx修改：**

```javascript
// 修改前（假的）
const handlePasswordChange = async (values) => {
  // TODO: 實作密碼變更 API
  console.log('變更密碼:', values);
  message.success('密碼變更成功');  // ← 假成功！
};

// 修改後（真的）
const handlePasswordChange = async (values) => {
  // 檢查新舊密碼不同
  if (values.oldPassword === values.newPassword) {
    message.error('新密碼不能與目前密碼相同');
    return;
  }

  setLoading(true);
  
  // 呼叫真實的API
  await api.post('/auth/change-password', {
    oldPassword: values.oldPassword,
    newPassword: values.newPassword
  });
  
  message.success('密碼變更成功，請使用新密碼重新登入');
  setPasswordModalVisible(false);
  passwordForm.resetFields();
};
```

## 測試步驟

### 測試1：變更密碼成功

1. 登入 admin@company.com（密碼：admin999）
2. 進入「設定」頁面
3. 點擊「變更密碼」
4. 輸入：
   - 目前密碼：admin999
   - 新密碼：admin99
   - 確認新密碼：admin99
5. 點擊「確認變更」
6. 應該看到：「密碼變更成功，請使用新密碼重新登入」
7. **登出**
8. 用舊密碼 admin999 登入 → **應該失敗**（401錯誤）
9. 用新密碼 admin99 登入 → **應該成功**

### 測試2：舊密碼錯誤

1. 登入後進入設定
2. 輸入錯誤的舊密碼
3. 應該看到：「目前密碼錯誤」

### 測試3：新密碼與舊密碼相同

1. 輸入相同的新舊密碼
2. 應該看到：「新密碼不能與目前密碼相同」

## 資料庫驗證

變更密碼後，可以檢查資料庫：

```sql
SELECT UserId, Email, PasswordHash 
FROM Users 
WHERE Email = 'admin@company.com';
```

PasswordHash 應該會改變（BCrypt hash值）。

## 安全性說明

1. **密碼加密**：使用 BCrypt 加密，無法反向解密
2. **JWT驗證**：必須登入（JWT Token有效）才能變更密碼
3. **舊密碼驗證**：防止未授權修改
4. **前後端雙重驗證**：新舊密碼不同的檢查
