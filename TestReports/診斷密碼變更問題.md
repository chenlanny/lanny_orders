# 密碼變更功能診斷指南

## 問題：送出密碼變更時顯示「密碼變更失敗」

## 診斷步驟

### 步驟1：確認後端是否已重啟 ⭐ 最可能的原因

**檢查方法：**
1. 查看運行 `dotnet run` 的終端機
2. 看看是否有以下類似的輸出：
   ```
   Now listening on: http://localhost:5000
   Application started.
   ```

**如果後端還沒重啟：**
1. 在運行 `dotnet run` 的終端機按 `Ctrl + C` 停止
2. 重新執行：
   ```powershell
   cd d:\github\ORDERS\A_order\database\backend
   dotnet run
   ```
3. 確認看到「Application started」訊息
4. 重新測試密碼變更功能

### 步驟2：檢查瀏覽器 Console 錯誤

**開啟方式：**
- Chrome/Edge: 按 `F12` → 點擊「Console」標籤

**查看錯誤訊息：**
- 是否有紅色錯誤訊息？
- 錯誤內容是什麼？

**常見錯誤：**

#### 錯誤1：404 Not Found
```
POST http://localhost:5000/api/auth/change-password 404
```
**原因：** 後端還沒重啟，舊版本沒有這個API端點
**解決：** 重啟後端（見步驟1）

#### 錯誤2：401 Unauthorized  
```
POST http://localhost:5000/api/auth/change-password 401
```
**原因：** JWT Token無效或過期
**解決：** 登出後重新登入

#### 錯誤3：400 Bad Request
```
{
  "success": false,
  "message": "目前密碼錯誤"
}
```
**原因：** 輸入的舊密碼不正確
**解決：** 確認輸入正確的舊密碼

#### 錯誤4：500 Internal Server Error
```
POST http://localhost:5000/api/auth/change-password 500
```
**原因：** 後端程式錯誤
**解決：** 查看後端 Console 的錯誤訊息

### 步驟3：檢查 Network 請求

**開啟方式：**
- 按 `F12` → 點擊「Network」標籤
- 勾選「Preserve log」
- 重新提交密碼變更
- 找到 `change-password` 請求

**檢查內容：**

1. **Request URL:** 
   ```
   http://localhost:5000/api/auth/change-password
   ```
   確認路徑正確

2. **Request Headers:**
   ```
   Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
   ```
   確認有 Token

3. **Request Payload:**
   ```json
   {
     "oldPassword": "admin999",
     "newPassword": "admin99"
   }
   ```
   確認資料格式正確

4. **Response:**
   - Status Code: 200 (成功) / 400, 404, 500 (失敗)
   - Response Body: 查看錯誤訊息

### 步驟4：檢查後端 Console 輸出

**查看內容：**
- 是否有錯誤訊息（紅色）？
- 是否有「密碼變更失敗」的 Log？

**常見後端錯誤：**

#### 錯誤1：ChangePasswordAsync 方法不存在
```
error CS1061: 'AuthService' does not contain a definition for 'ChangePasswordAsync'
```
**原因：** 後端沒有重新編譯
**解決：** 
```powershell
cd d:\github\ORDERS\A_order\database\backend
dotnet build
dotnet run
```

#### 錯誤2：資料庫連線失敗
```
SqlException: Cannot open database
```
**原因：** 資料庫連線問題
**解決：** 確認 SQL Server 正在運行

### 步驟5：手動測試 API

**使用命令行測試：**

```powershell
# 先登入取得 Token
$loginResponse = Invoke-RestMethod -Uri "http://localhost:5000/api/auth/login" -Method POST -Body (@{email="admin@company.com";password="admin999"} | ConvertTo-Json) -ContentType "application/json"
$token = $loginResponse.data.token

# 測試變更密碼
Invoke-RestMethod -Uri "http://localhost:5000/api/auth/change-password" -Method POST -Headers @{Authorization="Bearer $token"} -Body (@{oldPassword="admin999";newPassword="admin99"} | ConvertTo-Json) -ContentType "application/json"
```

**預期結果：**
```json
{
  "success": true,
  "message": "密碼變更成功",
  "data": null
}
```

## 最可能的原因排序

1. ⭐⭐⭐ **後端未重啟**（90%機率）
   - 解決：重啟後端

2. ⭐⭐ **輸入的舊密碼錯誤**（5%機率）
   - 解決：確認舊密碼正確

3. ⭐ **JWT Token過期**（3%機率）
   - 解決：重新登入

4. **其他錯誤**（2%機率）
   - 需要查看具體錯誤訊息
