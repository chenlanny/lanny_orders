# 外送費計算邏輯修復說明

## 問題描述
- **症狀**：揪團管理頁面顯示的外送費與分帳明細頁面計算的外送費不一致
- **影響**：財務計算錯誤，用戶分攤的外送費金額不正確

## 問題分析

### 資料結構

系統中有兩個地方儲存外送費：

1. **Event.DeliveryFee**（活動層級）
   - 儲存**所有店家外送費的總和**
   - 例如：店A $50 + 店B $50 = Event.DeliveryFee $100

2. **OrderGroup.DeliveryFee**（店家層級）
   - 儲存**各店家自己的外送費**
   - 例如：店A的 OrderGroup.DeliveryFee = $50

### 建立揪團時的邏輯

**EventService.cs - CreateEventAsync**：
```csharp
// 建立活動時，Event.DeliveryFee 設為所有店家外送費的總和
var newEvent = new Event
{
    // ...
    DeliveryFee = request.OrderGroups.Sum(g => g.DeliveryFee), // 總運費
    // ...
};

// 各店家的訂單群組保留自己的外送費
var orderGroups = request.OrderGroups.Select(groupDto => new OrderGroup
{
    // ...
    DeliveryFee = groupDto.DeliveryFee,  // 店家自己的外送費
    // ...
}).ToList();
```

### 錯誤的計算邏輯（修復前）

**SummaryService.cs - GetEventSummaryAsync**：

```csharp
foreach (var orderGroup in eventEntity.OrderGroups)
{
    // 錯誤：使用 Event 的總外送費來計算每人應分攤金額
    var deliveryFeePerPerson = participantCount > 0
        ? Math.Ceiling(eventEntity.DeliveryFee / participantCount)
        : 0;
    
    // ...
    
    storeSummaries.Add(new StoreSummaryDto
    {
        // ...
        DeliveryFee = eventEntity.DeliveryFee,  // ❌ 顯示總外送費
        // ...
        Total = storeOrders.Sum(o => o.SubTotal) + eventEntity.DeliveryFee,  // ❌ 錯誤計算
    });
}
```

### 問題範例

假設有2家店，各有不同的參與者：

**設定**：
- 店家A（50嵐）：外送費 $50，參與者 2 人（張三、李四）
- 店家B（清心）：外送費 $50，參與者 3 人（張三、王五、趙六）
- Event.DeliveryFee = $100（總和）

**錯誤計算（修復前）**：
```
店家A：
  每人外送費 = Ceiling(100 / 2) = $50  ❌ 錯誤！
  張三應付 = 品項 + $50
  李四應付 = 品項 + $50
  店家A總計 = 品項總計 + $100

店家B：
  每人外送費 = Ceiling(100 / 3) = $34  ❌ 錯誤！
  張三應付 = 品項 + $34
  王五應付 = 品項 + $34
  趙六應付 = 品項 + $34
  店家B總計 = 品項總計 + $100
```

問題：
1. 每家店都用總外送費 $100 計算
2. 張三在兩家店都要付外送費分攤，累計 $84
3. 總外送費變成 $200（應該只有 $100）

**正確計算（修復後）**：
```
店家A：
  每人外送費 = Ceiling(50 / 2) = $25  ✅ 正確！
  張三應付 = 品項 + $25
  李四應付 = 品項 + $25
  店家A總計 = 品項總計 + $50

店家B：
  每人外送費 = Ceiling(50 / 3) = $17  ✅ 正確！
  張三應付 = 品項 + $17
  王五應付 = 品項 + $17
  趙六應付 = 品項 + $17
  店家B總計 = 品項總計 + $51（因為無條件進位）
```

正確：
1. 各店用自己的外送費計算
2. 張三總外送費 = $25 + $17 = $42
3. 總外送費 = $101（$50 + $51，因為店B無條件進位）

## 解決方案

### 修改 1：使用正確的外送費來源

**檔案位置**：`database/backend/Services/SummaryService.cs`

**修改內容**：

**修改前**：
```csharp
// 計算每人外送費 - 使用活動總外送費（錯誤）
var deliveryFeePerPerson = participantCount > 0
    ? Math.Ceiling(eventEntity.DeliveryFee / participantCount)
    : 0;
```

**修改後**：
```csharp
// 計算每人外送費 - 使用此店家的外送費（正確）
var deliveryFeePerPerson = participantCount > 0
    ? Math.Ceiling(orderGroup.DeliveryFee / participantCount)
    : 0;
```

### 修改 2：店家總額計算

**修改前**：
```csharp
storeSummaries.Add(new StoreSummaryDto
{
    StoreName = store?.Name ?? "未知",
    StoreCategory = store?.Category ?? "未知",
    DeliveryFee = eventEntity.DeliveryFee,  // ❌ 顯示總外送費
    ParticipantCount = participantCount,
    Total = storeOrders.Sum(o => o.SubTotal) + eventEntity.DeliveryFee,  // ❌ 錯誤
    UserSummaries = userSummaries.OrderBy(u => u.UserName).ToList()
});
```

**修改後**：
```csharp
storeSummaries.Add(new StoreSummaryDto
{
    StoreName = store?.Name ?? "未知",
    StoreCategory = store?.Category ?? "未知",
    DeliveryFee = orderGroup.DeliveryFee,  // ✅ 顯示店家自己的外送費
    ParticipantCount = participantCount,
    Total = storeOrders.Sum(o => o.SubTotal) + orderGroup.DeliveryFee,  // ✅ 正確
    UserSummaries = userSummaries.OrderBy(u => u.UserName).ToList()
});
```

### 修改 3：總金額計算

**修改前**：
```csharp
var grandTotal = allOrders.Sum(o => o.SubTotal) + eventEntity.DeliveryFee;
```

**修改後**：
```csharp
// 計算所有店家的外送費總和
var totalDeliveryFee = eventEntity.OrderGroups.Sum(og => og.DeliveryFee);
var grandTotal = allOrders.Sum(o => o.SubTotal) + totalDeliveryFee;
```

## 架構說明

### 為何需要兩個 DeliveryFee？

**Event.DeliveryFee**（活動層級）：
- 用途：快速顯示整個活動的總外送費
- 儲存：所有店家外送費的總和
- 查詢效能：避免每次都要 JOIN OrderGroups

**OrderGroup.DeliveryFee**（店家層級）：
- 用途：分帳計算、各店家明細
- 儲存：各店家自己的外送費
- 靈活性：支援不同店家不同外送費

### 替代方案（未採用）

**方案 1：只保留 OrderGroup.DeliveryFee**
```csharp
// 移除 Event.DeliveryFee
// 需要時動態計算
var eventDeliveryFee = event.OrderGroups.Sum(og => og.DeliveryFee);
```
- 優點：數據一致性更好
- 缺點：每次查詢都需要 JOIN

**方案 2：同步更新兩個欄位**
```csharp
// 確保 Event.DeliveryFee 始終等於 OrderGroups.Sum(DeliveryFee)
// 使用 Database Trigger 或 Application Logic
```
- 優點：數據冗餘但一致
- 缺點：維護成本高

**當前方案（推薦）**：
- Event.DeliveryFee 用於顯示總計
- OrderGroup.DeliveryFee 用於分帳計算
- 明確區分用途，避免混用
