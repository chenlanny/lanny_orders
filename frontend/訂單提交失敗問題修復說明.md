# 訂單提交失敗問題修復說明

## 🔴 問題描述

**症狀**: 在揪團店家中選完品項後，點擊「確認送出訂單」按鈕失敗，訂單無法提交。

**影響**: 所有使用者無法下單，系統核心功能無法使用（Critical Bug）

---

## 🔍 問題分析

### 根本原因

前端傳送給後端的訂單資料格式**不正確**，特別是**配料（toppings）**欄位的格式不符合後端預期。

### 資料格式不匹配

**❌ 前端錯誤傳送的格式**：
```javascript
{
  groupId: 1,
  items: [{
    itemId: 75,
    sizeCode: "中杯",
    quantity: 2,
    toppings: [
      { name: "珍珠", price: 10 },   // 物件陣列（錯誤）
      { name: "椰果", price: 5 }
    ]
  }]
}
```

**✅ 後端預期的格式**：
```javascript
{
  groupId: 1,
  items: [{
    itemId: 75,
    sizeCode: "中杯",
    quantity: 2,
    toppings: ["珍珠", "椰果"]   // 字串陣列（正確）
  }]
}
```

### 為什麼會有這個問題？

1. **購物車儲存格式**：購物車為了顯示價格，儲存配料時包含 `{name, price}` 物件
2. **直接傳送**：提交訂單時直接將購物車的 toppings 傳給後端
3. **後端預期**：後端只需要配料名稱（字串陣列），會自己從菜單資料中查找價格
4. **格式不符**：導致後端驗證失敗或無法正確處理

## ✅ 解決方案

### 修改位置

**檔案**: `frontend/src/pages/EventDetail.jsx`  
**函數**: `handleSubmitOrder` (第 90 行)  
**修改行**: 第 119 行

### 修改內容

**修改前**（錯誤）:
```javascript
const orderData = {
  groupId: parseInt(groupId),
  items: groupItems.map(item => ({
    itemId: item.itemId,
    sizeCode: item.sizeCode,
    quantity: item.quantity,
    customOptions: item.customOptions || '',
    toppings: item.toppings || [],  // ❌ 直接傳物件陣列
    note: item.note || '',
  })),
};
```

**修改後**（正確）:
```javascript
const orderData = {
  groupId: parseInt(groupId),
  items: groupItems.map(item => ({
    itemId: item.itemId,
    sizeCode: item.sizeCode,
    quantity: item.quantity,
    customOptions: item.customOptions || '',
    toppings: item.toppings ? item.toppings.map(t => t.name) : [],  // ✅ 只傳名稱
    note: item.note || '',
  })),
};
```

### 關鍵修改

```javascript
// 從：物件陣列
toppings: item.toppings || []

// 改為：字串陣列（只取 name 屬性）
toppings: item.toppings ? item.toppings.map(t => t.name) : []
```

**轉換範例**：
```javascript
// 輸入：[{name: "珍珠", price: 10}, {name: "椰果", price: 5}]
// 輸出：["珍珠", "椰果"]
```

## 📊 後端處理邏輯

### 為什麼後端只需要配料名稱？

**安全性考量**：防止前端竄改價格

**處理流程**：

1. **接收訂單**：
   ```javascript
   // 前端只傳配料名稱
   toppings: ["珍珠", "椰果"]
   ```

2. **查找價格**（後端）：
   ```csharp
   // 從 MenuItem.Options 的 JSON 中查找實際價格
   var menuOptions = JsonSerializer.Deserialize<MenuOptions>(menuItem.Options);
   // menuOptions.toppings = [
   //   {name: "珍珠", price: 10},
   //   {name: "椰果", price: 5}
   // ]
   
   foreach (var toppingName in toppings) {
       var topping = menuOptions.toppings.FirstOrDefault(t => t.name == toppingName);
       totalPrice += topping.price;
   }
   ```

3. **計算總價**：
   ```
   基本價格 + 配料價格 (10 + 5) = 最終價格
   ```

**優點**：
- 🔒 價格由後端控制，無法被竄改
- ✅ 前端只負責選擇配料
- ✅ 後端根據菜單資料計算價格
