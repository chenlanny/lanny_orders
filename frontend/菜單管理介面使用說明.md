# 菜單管理介面使用說明

## 更新內容

已將菜單編輯介面從 JSON 格式改為更友善的表單介面，不需要懂程式也能輕鬆編輯菜單！

---

## 新介面功能說明

### 1. 基本資訊
- **品項名稱**：輸入飲料或餐點的名稱（例如：珍珠奶茶、炸雞便當）
- **類別**：輸入分類（例如：奶茶、綠茶、果茶、主餐）

---

### 2. 價格設定（支援多種尺寸）

**操作方式**：
- 點擊「新增價格選項」按鈕可以增加新的尺寸
- 每個價格選項包含：
  - **尺寸名稱**：例如「小杯」、「中杯」、「大杯」、「瓶裝」
  - **價格**：輸入金額，自動顯示「元」
- 點擊右側的 ❌ 圖示可以刪除該價格選項

**範例**：
```
小杯  40元
中杯  50元  
大杯  60元
```

---

### 3. 甜度選項

**操作方式**：
- 直接輸入甜度名稱，按 Enter 鍵新增
- 可以新增多個選項
- 點擊已選項目的 ✕ 可以刪除

**常用選項**：
- 正常
- 少糖
-半糖
- 微糖
- 無糖

---

### 4. 溫度選項

**操作方式**：
- 直接輸入溫度名稱，按 Enter 鍵新增
- 可以新增多個選項
- 點擊已選項目的 ✕ 可以刪除

**常用選項**：
- 正常冰
- 少冰
- 微冰
- 去冰
- 溫
- 熱

---

### 5. 配料選項（可加價）

**操作方式**：
- 點擊「新增配料選項」按鈕增加新配料
- 每個配料包含：
  - **配料名稱**：例如「珍珠」、「椰果」、「布丁」
  - **加價金額**：輸入加價金額，自動顯示「元」
- 點擊右側的 ❌ 圖示可以刪除該配料

**範例**：
```
珍珠   10元
椰果   5元
布丁   15元
```

**備註**：如果配料不加價，將加價金額設為 0 元即可

---

### 6. 上架狀態

使用開關切換：
- **上架**（綠色）：客人可以看到並訂購
- **下架**（灰色）：客人看不到此品項

## 注意事項

1. **至少設定一個價格**：每個品項必須至少有一個價格選項
2. **金額只能輸入數字**：系統會自動檢查並防止輸入錯誤格式
3. **甜度和溫度是選填**：如果品項不需要（如便當），可以留空
4. **配料可以設為 0 元**：表示免費提供該配料
5. **儲存前請檢查**：確認所有資訊正確再點擊「確定」

## 常見問題 Q&A

**Q1: 如何增加更多尺寸？**  
A: 點擊「價格設定」區域下方的「新增價格選項」按鈕即可。

**Q2: 配料可以不加價嗎？**  
A: 可以！將加價金額設為 0 元即可。

**Q3: 如果不需要甜度或溫度選項怎麼辦？**  
A: 這兩個欄位是選填的，可以留空不填。

**Q4: 可以修改已有的菜單嗎？**  
A: 可以！點擊菜單列表中的「編輯」按鈕即可修改。

**Q5: 如何刪除不要的選項？**  
A: 點擊該選項右側的 ❌ 圖示即可刪除。

## 技術說明（給開發者）

### 資料轉換

介面會自動將友善的表單格式轉換為後端所需的 JSON 格式：

**表單格式**：
```javascript
prices: [
  { sizeName: '中杯', price: 50 },
  { sizeName: '大杯', price: 60 }
]
```

**轉換為 JSON**：
```json
{
  "priceRules": "{\"中杯\":50,\"大杯\":60}",
  "options": "{\"sweetness\":[...],\"temperature\":[...],\"toppings\":[...]}"
}
```
