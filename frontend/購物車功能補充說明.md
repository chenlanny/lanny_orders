# 購物車功能補充說明

## 問題描述

使用者反映：點選「購物車」後畫面與首頁相同，懷疑沒有此功能。

## 問題分析

**原因**：
1. Header 元件中有「購物車」按鈕 (`onClick={() => navigate('/cart')}`)
2. 但 App.jsx 的路由配置中沒有定義 `/cart` 路由
3. 導致點擊後被 `Route path="*"` 重定向到 `/dashboard`

**技術細節**：
- ShoppingCart.jsx 元件本身已經實作完整
- cartStore 狀態管理也正常運作
- 只是缺少獨立的頁面和路由配置

## 解決方案

### 1. 創建購物車頁面

**檔案**: `frontend/src/pages/Cart.jsx`

**功能**：
- 使用 ShoppingCart 元件顯示商品列表
- 提供「返回」、「清空購物車」、「繼續購物」按鈕
- 顯示購物車為空時的友善提示
- 提醒使用者購物車內容需在活動頁面提交

**介面設計**：
```jsx
<Card>
  <Header>
    [← 返回] 🛒 我的購物車 [清空購物車]
  </Header>
  
  <Content>
    {items.length === 0 ? (
      <EmptyState />  // 空購物車提示 + 開始點餐按鈕
    ) : (
      <>
        <ShoppingCart />  // 商品列表
        <Alert>提醒尚未提交訂單</Alert>
        <Actions>
          [繼續購物]
        </Actions>
      </>
    )}
  </Content>
</Card>
```

### 2. 更新路由配置

**檔案**: `frontend/src/App.jsx`

**修改**：
```jsx
// 1. 匯入 Cart 元件
import Cart from './pages/Cart';

// 2. 添加路由
<Route path="cart" element={<Cart />} />
```

## 使用說明

### 購物流程

1. **加入商品**
   - 在活動詳情頁選擇店家
   - 選擇商品並設定選項
   - 點擊「加入購物車」

2. **查看購物車**
   - 點擊 Header 的「購物車」按鈕（有商品數量提示）
   - 進入購物車頁面

3. **管理商品**
   - 調整數量：使用 +/- 或直接輸入
   - 刪除商品：點擊垃圾桶圖示
   - 清空購物車：點擊「清空購物車」按鈕

4. **完成訂單**
   - 點擊「返回」或「繼續購物」回到活動頁面
   - 在活動頁面點擊「提交訂單」

### 購物車功能說明

#### 已實作功能 ✅
- 商品列表顯示（品項、尺寸、選項、加料、備註）
- 數量調整 (1-99)
- 單項商品刪除
- 清空購物車
- 即時計算小計和總計
- 空購物車提示
- 購物車商品數量提示 (Header 紅色圓圈)

#### 注意事項 ⚠️
- 購物車內容**僅儲存在瀏覽器**中（localStorage）
- 購物車商品**尚未送出訂單**
- 需要在**活動詳情頁面**點擊「提交訂單」才會真正下單
- 刷新頁面後購物車內容會保留
- 清除瀏覽器資料會清空購物車

1. **購物車持久化**：考慮同步到後端，避免換裝置後遺失

3. 沒有檢查活動是否已截止
4. 跨活動的商品會混在一起（建議未來區分）
