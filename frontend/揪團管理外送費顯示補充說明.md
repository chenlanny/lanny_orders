# 揪團管理頁面 - 外送費顯示功能補充說明

## 問題描述
- **症狀**：在「分帳明細」頁面可以看到店家外送費用，但在「揪團管理」頁面的店家列表中看不到外送費用
- **影響**：管理員無法在揪團管理介面中驗證外送費是否正確設定

## 根本原因分析

### 問題源頭
在 EventManagement.jsx 中，展開行顯示店家資訊時，欄位定義缺少「外送費」欄位。

**修改前的欄位定義**：
```javascript
const columns = [
  { title: '店家', dataIndex: 'storeName' },
  { title: '類別', dataIndex: 'category' },
  { title: '截止時間', dataIndex: 'deadline' },  // ❌ 缺少外送費
  { title: '狀態', dataIndex: 'status' },
  { title: '操作', key: 'action' }
];
```

### 後端資料驗證
後端 `OrderGroupSimpleDto` 確實包含外送費欄位：
```csharp
public class OrderGroupSimpleDto
{
    public int GroupId { get; set; }
    public int StoreId { get; set; }
    public string StoreName { get; set; } = string.Empty;
    public string Category { get; set; } = string.Empty;
    public DateTime Deadline { get; set; }
    public decimal DeliveryFee { get; set; }  // ✅ 後端有提供
    public string Status { get; set; } = string.Empty;
}
```

### 使用者需求
管理員需要在「揪團管理」頁面中：
1. 查看每個店家的外送費設定
2. 驗證外送費金額是否正確
3. 對照「分帳明細」中的外送費是否一致

## 解決方案

### 修改 1：店家列表新增外送費欄位

**檔案位置**：`frontend/src/pages/EventManagement.jsx`

**修改內容**：
```javascript

    {

    },
    {

      key: 'deadline',
      render: (deadline) => dayjs(deadline).format('YYYY-MM-DD HH:mm'),
    },
    // ...其他欄位

  return (

      columns={columns}
      dataSource={event.orderGroups}

      rowKey="groupId"
    />
  );
```

## UI/UX 改進

### 欄位排序邏輯
將「外送費」放在「類別」之後、「截止時間」之前的原因：
1. **店家識別**：店家名稱 → 類別（基本資訊）
2. **費用資訊**：外送費（重要財務資訊）
3. **時間狀態**：截止時間 → 狀態（時效性資訊）
4. **操作**：提早截止（動作）

這樣的順序符合「先了解是什麼店 → 費用多少 → 什麼時候截止 → 如何操作」的認知流程。

### 金額格式化
使用 `$${deliveryFee}` 格式化的好處：
- 清楚標示為金額
- 與分帳明細頁面格式一致
- 視覺上易於識別

### 可選的 UI 增強
如果未來需要更突出外送費資訊：

**方案 1：高外送費警示**
```javascript
render: (deliveryFee) => {
  const isHigh = deliveryFee >= 100;
  return (
    <span style={{ color: isHigh ? '#ff4d4f' : 'inherit', fontWeight: isHigh ? 'bold' : 'normal' }}>
      ${deliveryFee}
    </span>
  );
}
```

**方案 2：免外送費標籤**
```javascript
render: (deliveryFee) => {
  return deliveryFee === 0 
    ? <Tag color="green">免外送費</Tag>
    : `$${deliveryFee}`;
}
```
