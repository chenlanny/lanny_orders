# 分帳明細功能修復說明

## 問題描述
- **症狀**：送出訂單成功，但「分帳明細」頁面無資料顯示
- **原因**：前端期待的資料結構與後端返回的 DTO 不匹配

## 根本原因分析

### 前端期待的資料結構
```javascript
{
  title: "活動標題",
  status: "Open",
  createdAt: "2026-04-03T10:00:00",
  closedAt: null,
  grandTotal: 500,
  storeSummaries: [
    {
      storeName: "店家名稱",
      storeCategory: "Drink",
      deliveryFee: 50,
      participantCount: 5,
      total: 500,
      userSummaries: [...]
    }
  ],
  userTotals: [
    {
      userId: 1,
      userName: "使用者",
      department: "部門",
      itemTotal: 100,
      deliveryFeeTotal: 10,
      totalDue: 110
    }
  ]
}
```

### 後端原本返回的結構
```csharp
{
  EventName: "...",          // 不是 title
  OrderGroups: [...],        // 不是 storeSummaries
  UserBills: [...],          // 不是 userTotals
  // 缺少 grandTotal, createdAt, closedAt 等欄位
}
```

## 解決方案

### 1. 修改後端 DTO（OrderDtos.cs）
- 重新定義 `EventSummaryDto` 以符合前端需求
- 新增 `StoreSummaryDto` - 店家訂單摘要
- 新增 `UserStoreSummaryDto` - 用戶在某店家的訂單摘要
- 新增 `OrderItemSummaryDto` - 訂單項目摘要
- 新增 `ToppingDto` - 配料資訊
- 新增 `UserTotalDto` - 用戶總計

### 2. 重寫 SummaryService.cs
- 完全重構 `GetEventSummaryAsync` 方法
- 按店家分組計算外送費分攤
- 為每位用戶計算在各店家的小計
- 彙總用戶在所有店家的總計
- 解析配料資訊（使用 ToppingPrice 計算平均價格）

### 3. 關鍵邏輯

#### 外送費分攤計算
```csharp
// 每家店鋪的參與人數
var storeParticipants = storeOrders.Select(o => o.UserId).Distinct().ToList();
var participantCount = storeParticipants.Count;

// 每人外送費（無條件進位）
var deliveryFeePerPerson = participantCount > 0
    ? Math.Ceiling(eventEntity.DeliveryFee / participantCount)
    : 0;
```

#### 配料價格計算
```csharp
// 配料總價除以配料數量，得到平均價格
var averagePrice = toppingNames.Count > 0 
    ? order.ToppingPrice / toppingNames.Count 
    : 0;
```

## 測試步驟

1. **前置條件**：確保已有測試訂單
   - 登入系統
   - 加入商品到購物車
   - 提交訂單成功

2. **測試分帳明細**：
   - 進入活動頁面
   - 點擊「查看分帳明細」按鈕
   - 確認資料正確顯示：
     - ✓ 活動標題、狀態、時間
     - ✓ 總金額
     - ✓ 店家明細（店名、類別、外送費、參與人數）
     - ✓ 用戶訂單詳情（品項、配料、小計）
     - ✓ 外送費分攤計算
     - ✓ 用戶總結表（所有店家合計）

3. **驗證計算邏輯**：
   - 外送費 = 原外送費 ÷ 參與人數（無條件進位）
   - 用戶總計 = 品項總計 + 外送費總計
   - 總金額 = 所有訂單品項 + 外送費

## API 端點
- **GET** `/api/events/{eventId}/summary`
- **回應格式**：`ApiResponse<EventSummaryDto>`
