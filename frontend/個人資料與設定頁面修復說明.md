# 個人資料與設定頁面修復說明

## 問題描述
- **症狀**：點擊右上角管理員選單中的「個人資料」和「設定」選項後，頁面跳轉到首頁
- **原因**：路由配置缺失，沒有定義 `/profile` 和 `/settings` 路由

## 根本原因分析

### 前端路由問題
Header.jsx 的下拉選單中定義了導航動作：
```javascript
const userMenuItems = [
  {
    key: 'profile',
    label: '個人資料',
    onClick: () => navigate('/profile'),  // 路由不存在
  },
  {
    key: 'settings',
    label: '設定',
    onClick: () => navigate('/settings'),  // 路由不存在
  },
  // ...
];
```

但在 App.jsx 中沒有定義這兩個路由，導致：
1. 點擊後 React Router 找不到匹配的路由
2. 被 `<Route path="*">` 攔截
3. 重定向到 `/dashboard`（首頁）

## 解決方案

### 1. 建立 Profile.jsx 頁面
**檔案位置**：`frontend/src/pages/Profile.jsx`

**功能特色**：
- ✅ 顯示用戶基本資訊（姓名、Email、電話、部門）
- ✅ 頭像上傳功能（含檔案類型和大小驗證）
- ✅ 個人資料編輯表單
- ✅ 顯示帳號資訊（使用者 ID、角色、帳號狀態）
- ✅ 表單驗證（Email 格式、必填欄位）

**主要組件**：
```javascript
- Avatar 顯示區域（100px）
- Upload 頭像上傳（限制 2MB、圖片格式）
- Form 個人資料表單
  - 姓名（必填）
  - Email（必填、Email 格式驗證）
  - 電話（選填）
```

### 2. 建立 Settings.jsx 頁面
**檔案位置**：`frontend/src/pages/Settings.jsx`

**功能特色**：
- ✅ 安全性設定
  - 密碼變更（Modal 彈窗）
  - 舊密碼驗證

  - 訂單建立通知（開關）

  - 送達通知（開關）
- ✅ 介面設定
  - 語言選擇（繁中/簡中/英文）
  - 每頁顯示筆數（10/20/50/100）

**主要組件**：
```javascript
- 安全性設定卡片
  - 變更密碼按鈕 → 觸發 Modal
- 通知設定卡片
  - 3 個 Switch 開關（即時儲存）
- 介面設定卡片
  - 語言下拉選單
  - 每頁筆數下拉選單
  - 儲存按鈕
- 密碼變更 Modal
  - 目前密碼輸入
  - 新密碼輸入（含強度驗證）
  - 確認密碼輸入（一致性驗證）
```

### 3. 更新 App.jsx 路由配置

**新增導入**：
```javascript
import Profile from './pages/Profile';
import Settings from './pages/Settings';
```

**新增路由**：
```javascript
<Route path="profile" element={<Profile />} />
<Route path="settings" element={<Settings />} />
```

**路由保護**：
- 這兩個路由都在 `<ProtectedRoute>` 內
- 需要登入才能訪問
- 未登入會自動重定向到 `/login`

### 1. 後端 API 實作

**個人資料 API**：
```typescript
// 取得個人資料
GET /api/users/profile
Response: {
  userId: number,
  name: string,
  email: string,
  phone: string,
  department: string,
  role: string
}

// 更新個人資料
PUT /api/users/profile
Request: {
  name: string,
  email: string,
  phone?: string,
  department?: string
}

// 上傳頭像
POST /api/users/avatar
Content-Type: multipart/form-data
Request: { avatar: File }
Response: { avatarUrl: string }
```
