# 菜單編輯介面改版說明

## 📋 改版摘要

已將菜單管理介面從「JSON 編輯器」改為「友善表單介面」，讓不懂程式的使用者也能輕鬆編輯菜單！

---

## 🎯 改善重點

### 之前（JSON 格式）❌
```
價格規則 (JSON)
{"中杯":50,"大杯":60}

選項 (JSON)
{"sweetness":["正常","少糖","半糖"],"temperature":["正常冰","少冰"]}
```
**問題**：
- 需要懂 JSON 語法
- 容易打錯括號、引號
- 不適合非技術人員

## 🚀 使用方法

### 新增菜單步驟

1. **選擇店家** - 從下拉選單選擇要編輯的店家
2. **點擊「新增菜單」** - 開啟編輯視窗
3. **填寫基本資訊**
   - 品項名稱：例如「珍珠奶茶」
   - 類別：例如「奶茶系列」

4. **設定價格**
   - 預設會有「中杯」和「大杯」兩個選項
   - 可點擊「新增價格選項」增加更多尺寸
   - 可點擊 ❌ 刪除不要的尺寸
   - 只能輸入數字，系統會自動驗證

5. **設定甜度**（選填）
   - 在輸入框輸入甜度名稱，例如「正常」
   - 按 Enter 鍵新增
   - 點擊標籤上的 ✕ 可刪除

6. **設定溫度**（選填）
   - 在輸入框輸入溫度名稱，例如「正常冰」
   - 按 Enter 鍵新增
   - 點擊標籤上的 ✕ 可刪除

7. **設定配料**（選填）
   - 點擊「新增配料選項」
   - 輸入配料名稱和加價金額
   - 免費配料設為 0 元
   - 可點擊 ❌ 刪除

8. **設定上架狀態**
   - 切換開關：綠色=上架，灰色=下架

9. **點擊「確定」儲存**

## 🔧 技術細節

### 修改的檔案
- `frontend/src/pages/MenuManagement.jsx`

### 新增的元件
- `InputNumber` - 數字輸入框
- `Form.List` - 動態表單列表
- `Select mode="tags"` - 標籤選擇器
- `Row`, `Col` - 響應式佈局
- `Divider` - 區域分隔線

### 資料處理
- 提交時：表單 → JSON 格式 → API
- 編輯時：JSON 格式 → 表單 → 顯示

### 驗證規則
- 品項名稱：必填
- 類別：必填
- 至少一個價格：必填
- 甜度/溫度：選填
- 配料：選填

## ❓ 常見問題

**Q: 可以設定幾種尺寸？**  
A: 不限！可以根據需求新增任意數量的尺寸。

**Q: 配料一定要加價嗎？**  
A: 不一定！加價金額設為 0 元表示免費。

**Q: 便當類不需要甜度和溫度怎麼辦？**  
A: 這兩個欄位是選填的，直接留空即可。

**Q: 如何修改已有的菜單？**  
A: 點擊菜單列表中的「編輯」按鈕，系統會自動載入現有資料到表單中。

**Q: 誤刪了某個選項怎麼辦？**  
A: 如果還沒點「確定」，直接點「取消」即可。如果已儲存，重新編輯再加回來即可。
